# PDFタイトル一括リネームツール 仕様書

| 項目 | 内容 |
|---|---|
| 製品名 | PDFタイトル一括リネーム AI版 |
| バージョン | 0.1.4 |
| 対象OS | Windows 10（19041以降）／Windows 11、x64 |
| GUI | WPF／.NET 10 |
| OCR | Windows標準OCR |
| AI連携 | Codex App Server（ChatGPTログイン） |
| 文書日付 | 2026-07-15 |

## 1. 目的

ファイル名やPDFメタデータが正しく設定されていないPDFについて、先頭2ページの見た目とOCR結果から正式な文書タイトルを推定し、利用者の確認後に選択したファイルだけを安全に一括リネームする。

本製品は、PDFメタデータからファイル名を生成する [PDF Title to Filename（非AI版）](https://github.com/fa-yoshinobu/PDF_Title_to_Filename) を補完するAI版と位置づける。Titleなどのメタデータが信頼できる場合は非AI版、メタデータが空・不正確なPDFやスキャンPDFは本製品を使用する。

## 2. 対象範囲

### 2.1 対象

- 利用者が選択した1フォルダ直下のPDF
- テキストPDFおよびスキャンPDF
- 日本語を中心とする文書
- 文書タイトルが先頭2ページ内に存在するPDF

### 2.2 対象外

- サブフォルダ内のPDF
- PDF本文やメタデータの書き換え
- PDFの結合、分割、圧縮
- パスワード付き・暗号化PDFの解除
- AIが推定した名前による無確認の自動リネーム
- 完全オフラインでのAI判定

## 3. システム構成

```text
選択フォルダ直下のPDF
        ↓
先頭2ページを画像として描画（白背景）
        ↓
Windows OCR（文字列・座標・大きさ）
        ↓
ページ画像＋OCRレイアウトをCodex App Serverへ送信
        ↓
タイトル・確信度・判断根拠を構造化JSONで取得
        ↓
画面で確認・編集・対象選択
        ↓
確認ダイアログ後に安全な一括リネーム
```

Codex App Serverはローカルの標準入出力を使用する。AI推論時にはOpenAIへの通信が発生する。OpenAI PlatformのAPIキーは使用せず、各利用者がCodex CLIからChatGPTアカウントへログインする。

## 4. 機能要件

### F-001 フォルダ選択

- Windowsのフォルダ選択ダイアログを表示する。
- 選択フォルダ直下だけを検索する。
- 拡張子の大文字・小文字を区別せず `.pdf` を列挙する。
- 初期状態では列挙した全PDFを対象として選択する。

### F-002 対象選択

- PDFごとにチェックボックスを表示する。
- 「すべて選択」「選択解除」を提供する。
- OCR・AI判定およびリネームは、その時点で選択されているPDFを対象とする。

### F-003 PDFページ描画

- 各PDFの先頭から最大2ページを処理する。
- PDFページを画像として描画する。
- 基本描画倍率は2.4倍とする。
- 描画幅は最大2400ピクセルとする。
- 透明背景のPDFでもOCRできるよう、描画背景を白色に固定する。
- 生成画像は一時フォルダに保存し、処理終了後に削除を試みる。

### F-004 OCR

- Windows標準OCRを端末内で実行する。
- 日本語OCRが利用できる場合は `ja-JP` を優先する。
- 日本語OCRが利用できない場合は、ユーザープロファイルで利用可能なOCR言語を使用する。
- OCR結果として、行の文字列および各単語の矩形座標を取得する。
- OCRエンジンを初期化できない場合は、Windowsの言語機能追加を案内する。

### F-005 AIタイトル判定

- Codex App Serverを標準入出力のJSON-RPCで起動する。
- スレッドは一時スレッドとし、サンドボックスを読み取り専用、承認方式を `never`、ネットワークツールを無効にする。
- PDFごとに独立したAI判定を行い、別PDFの内容を混在させない。
- AI入力には次の情報だけを使用する。
  - 先頭2ページのページ画像
  - OCR行の文字列
  - OCR行のX座標、Y座標、幅、高さ
  - ページ番号およびページ寸法
- 次の情報はタイトル判断に使用しない。
  - PDFタイトルプロパティ
  - 作成者、件名、キーワードなどのPDFメタデータ
  - 現在のファイル名
- AIには、ページ上部、中央配置、大きい文字、独立見出しを重視させる。
- 会社名、日付、版番号、文書番号、ヘッダー、フッターは原則タイトルから除外させる。
- OCR誤認はページ画像と文脈から補正させる。
- 固有名詞を根拠なく創作させない。

### F-006 AI出力

AI出力は次のJSON形式に制約する。

```json
{
  "title": "推定タイトル",
  "confidence": 0.92,
  "reason": "表紙中央の最大文字と副題から判断"
}
```

- `title`: 1～160文字。拡張子を含めない。
- `confidence`: 0以上1以下。
- `reason`: 1～300文字。
- 画面上では確信度をパーセント表示する。

### F-007 結果確認・編集

- 一覧に次を表示する。
  - 対象チェック
  - 現在のファイル名
  - 変更後のファイル名
  - AI確信度
  - AIの判断根拠
  - 処理状態
- 変更後のファイル名は利用者が編集できる。
- AI判定だけで自動的にリネームしない。

### F-008 ファイル名正規化

- AIタイトルまたは利用者編集値をUnicode NFKCで正規化する。
- 入力末尾の `.pdf` は一度除去し、最終的に1つだけ付加する。
- Windowsで使用できない文字および制御文字を空白へ置換する。
- 連続空白を1文字へまとめる。
- 末尾と先頭の空白・ピリオドを除去する。
- 空文字になった場合は `無題.pdf` とする。
- `CON`、`PRN`、`AUX`、`NUL`、`COM1`～`COM9`、`LPT1`～`LPT9` の予約名には先頭に `_` を付ける。
- タイトル本体は原則最大140文字とし、フォルダパス長に応じて短縮する。

### F-009 重複回避

- 既存ファイルを上書きしない。
- 同名候補が存在する場合は ` (2)`、` (3)` の形式で連番を付加する。
- 同一バッチ内で同じタイトル候補が複数ある場合も連番を付加する。
- リネーム直前にも対象ファイルと変更先の存在を再確認する。

### F-010 一括リネーム

- 選択済みかつ変更後ファイル名が入力済みのPDFだけを対象とする。
- 実行前に元ファイル名と変更後ファイル名のプレビューを表示する。
- 利用者が明示的に「はい」を選択した場合だけ実行する。
- 各ファイルの成功・失敗を個別に記録する。
- 一部失敗時も、他のファイルの処理を継続する。

### F-011 ログイン

- 起動時に `codex login status` でCodex CLIとログイン状態を確認する。
- 未ログイン時は「ChatGPTにログイン」ボタンを表示する。
- ログインには `codex login` を使用する。
- 認証情報をアプリ自身で読み取り・保存・配布しない。

### F-012 進捗と中止

- OCR中、AI判定中、確認待ち、解析失敗、リネーム済みなどの状態を表示する。
- 全体の進捗をプログレスバーで表示する。
- OCR・AI判定中は中止操作を提供する。
- 1件の解析失敗でバッチ全体を停止せず、次のPDFへ進む。

## 5. 画面仕様

### 5.1 ヘッダー

- 製品名と処理概要を表示する。
- Codex CLIの状態を「ログイン済み」「未ログイン」「CLIなし」で表示する。
- ChatGPTログインボタンを配置する。

### 5.2 フォルダ選択領域

- 選択中のフォルダを読み取り専用テキストボックスに表示する。
- フォルダ選択ボタンを配置する。

### 5.3 PDF一覧

- DataGrid形式で表示する。
- 変更後ファイル名列だけを編集可能とする。
- その他の結果列は読み取り専用とする。

### 5.4 操作領域

- 処理状態テキスト
- プログレスバー
- 中止ボタン
- 「OCR＋AIでタイトル推定」ボタン
- 「選択したPDFをリネーム」ボタン

## 6. 非機能要件

### 6.1 安全性

- 既存ファイルを上書きしない。
- AIにファイル変更権限を与えず、実ファイルのリネームはGUIアプリ自身が確認後に行う。
- Codex App Serverのスレッドは読み取り専用とする。
- AIにツール実行、ファイル探索、Web検索を行わないよう指示する。
- 認証ファイルを直接操作しない。

### 6.2 プライバシー

- OCR処理はWindows端末内で実行する。
- AI判定時は、ページ画像およびOCRレイアウト情報がCodex経由でOpenAIへ送信される。
- 現在のファイル名とPDFメタデータはAIへ送信しない。
- 機密文書を処理する場合は、利用組織のOpenAI利用規則に従う。

### 6.3 可用性

- AI判定にはインターネット接続、Codex CLI、ChatGPTログインが必要である。
- OCR自体はオフラインで動作する。
- AI判定に失敗してもPDF原本を変更しない。

### 6.4 配布

- `win-x64` 向け自己完結型単一EXEとして発行する。
- .NETランタイムの別途インストールを不要とする。
- Codex CLIは別途インストールおよびログインが必要である。

## 7. エラー処理

| エラー | 動作 |
|---|---|
| Codex CLIなし | ヘッダーに表示し、AI判定を無効化する |
| Codex未ログイン | ログインを案内し、AI判定を開始しない |
| OCR言語なし | Windows言語設定でOCR機能を追加するよう表示する |
| PDF破損・暗号化 | 対象行を解析失敗にし、次のPDFへ進む |
| AI応答不正 | 対象行を解析失敗にし、PDF原本を変更しない |
| 元ファイル消失 | リネーム失敗として記録する |
| 変更先競合 | 上書きせず失敗として記録する |
| 利用者による中止 | 現在処理をキャンセルし、未処理PDFを変更しない |

## 8. 制約事項

- タイトル判定対象は先頭2ページに限定する。
- OCR精度は原稿解像度、傾き、汚れ、文字方向、Windowsに導入されたOCR言語に依存する。
- Codexの判断は確率的であり、正式タイトルを保証しない。実行前の利用者確認を必須とする。
- 本バージョンはリネーム結果の自動Undo履歴を保持しない。
- 複数ファイルのリネーム中に一部失敗した場合、成功済みファイルは自動では元に戻さない。
- x64 Windows専用とする。

## 9. 受入基準

1. 選択フォルダ直下のPDFだけが一覧に表示されること。
2. 大文字拡張子 `.PDF` も一覧に表示されること。
3. 先頭2ページが白背景画像として描画されること。
4. 日本語の表紙タイトルと2ページ目本文をOCRできること。
5. AI入力にPDFメタデータと現在のファイル名が含まれないこと。
6. AIからタイトル、確信度、判断根拠を取得できること。
7. 候補名を画面上で編集できること。
8. 未選択PDFがリネームされないこと。
9. 実行前に変更内容の確認ダイアログが表示されること。
10. 既存ファイルを上書きせず、重複時に連番が付くこと。
11. Windows禁止文字と予約名が安全化されること。
12. Codex未ログイン時にAI判定が実行されないこと。
13. 自己完結EXEが起動すること。
